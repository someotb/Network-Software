# Security Audit Checklist для photos-s15

## Проект
- **Название**: photos-s15
- **Сервис**: photos-svc-s15
- **Порт**: 8155
- **Дополнительное поле**: url (str)

## Чек-лист безопасности (OWASP Top 10 2021)

### 1. A01:2021 – Broken Access Control
- [ ] Проверка авторизации на всех эндпоинтах
- [ ] Валидация прав доступа к фотографиям (пользователь может видеть только свои фото)
- [ ] Защита от IDOR (Insecure Direct Object Reference)
- [ ] Проверка rate limiting для предотвращения перебора ID

### 2. A02:2021 – Cryptographic Failures
- [ ] Использование HTTPS для всех соединений
- [ ] Хранение паролей с использованием bcrypt/Argon2
- [ ] Шифрование чувствительных данных в базе
- [ ] Безопасное хранение API ключей и секретов
- [ ] Проверка сертификатов SSL/TLS

### 3. A03:2021 – Injection
- [ ] Защита от SQL Injection (использование параметризованных запросов)
- [ ] Защита от NoSQL Injection
- [ ] Валидация и санитизация всех входных данных
- [ ] Защита от Command Injection при обработке файлов
- [ ] Использование ORM с prepared statements

### 4. A04:2021 – Insecure Design
- [ ] Наличие rate limiting на API
- [ ] Ограничение размера загружаемых файлов
- [ ] Валидация типов файлов (только изображения)
- [ ] Защита от DoS атак
- [ ] Логирование критических операций

### 5. A05:2021 – Security Misconfiguration
- [ ] Отключение debug режима в production
- [ ] Удаление дефолтных credentials
- [ ] Настройка CORS политики
- [ ] Отключение directory listing
- [ ] Безопасные HTTP заголовки (CSP, X-Frame-Options, etc.)
- [ ] Актуальные версии зависимостей

### 6. A06:2021 – Vulnerable and Outdated Components
- [ ] Регулярное обновление зависимостей
- [ ] Сканирование уязвимостей (pip-audit, safety)
- [ ] Использование проверенных библиотек
- [ ] Мониторинг CVE для используемых компонентов

### 7. A07:2021 – Identification and Authentication Failures
- [ ] Надёжная аутентификация (JWT, OAuth2)
- [ ] Защита от brute-force атак
- [ ] Безопасное хранение сессий
- [ ] Валидация и истечение токенов
- [ ] Multi-factor authentication (опционально)

### 8. A08:2021 – Software and Data Integrity Failures
- [ ] Проверка целостности загружаемых файлов
- [ ] Валидация подписей образов Docker
- [ ] Использование checksum для файлов
- [ ] Защита CI/CD pipeline
- [ ] Code signing

### 9. A09:2021 – Security Logging and Monitoring Failures
- [ ] Логирование всех попыток доступа
- [ ] Логирование ошибок аутентификации
- [ ] Мониторинг подозрительной активности
- [ ] Алерты на критические события
- [ ] Защита логов от модификации

### 10. A10:2021 – Server-Side Request Forgery (SSRF)
- [ ] Валидация URL при загрузке фото по ссылке
- [ ] Whitelist разрешённых доменов
- [ ] Блокировка внутренних IP адресов (127.0.0.1, 192.168.x.x)
- [ ] Timeout для внешних запросов
- [ ] Проверка Content-Type ответов

## Дополнительные проверки для сервиса фотографий

### Безопасность загрузки файлов
- [ ] Проверка MIME типа файла
- [ ] Проверка расширения файла
- [ ] Проверка magic bytes (сигнатуры файла)
- [ ] Ограничение размера файла (например, max 10MB)
- [ ] Сканирование на вирусы/malware
- [ ] Генерация уникальных имён файлов
- [ ] Хранение файлов вне web root

### Обработка изображений
- [ ] Защита от image bombs (decompression bombs)
- [ ] Валидация размеров изображения
- [ ] Удаление EXIF метаданных (могут содержать GPS координаты)
- [ ] Использование безопасных библиотек (Pillow с актуальными патчами)

### API Security
- [ ] Использование API ключей
- [ ] Rate limiting (например, 100 req/min на пользователя)
- [ ] Input validation на всех эндпоинтах
- [ ] Output encoding для предотвращения XSS
- [ ] Версионирование API

### Infrastructure Security
- [ ] Изоляция контейнеров
- [ ] Минимальные привилегии для сервисов
- [ ] Network policies в Kubernetes
- [ ] Secrets management (не хардкодить в коде)
- [ ] Regular security updates
