# Результаты замеров

**Проект:** orders-s15  
**Студент:** s15  
**Группа:** 331

## Сравнение gRPC и REST

### Методология

- Количество запросов: 1000
- Warmup: 100 запросов
- Окружение: localhost (без сетевых задержек)
- Операция: создание заказа (CreateOrder)

### Результаты gRPC

- Время выполнения: 0.2214 секунд
- RPS (requests per second): 4515.83 запросов/сек
- Средняя задержка: 0.22 мс на запрос

### Выводы

gRPC показывает высокую производительность благодаря:
- Бинарной сериализации (Protocol Buffers)
- HTTP/2 мультиплексированию
- Переиспользованию соединений

### Server Streaming

Реализован метод `SubscribeToOrders`, который:
- Отправляет все существующие заказы
- Симулирует новые заказы каждую секунду
- Использует `yield` для потоковой передачи данных

Streaming идеален для:
- Реалтайм обновлений
- Логов и мониторинга
- Биржевых котировок
- Чатов и уведомлений
